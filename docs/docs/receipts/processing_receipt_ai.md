# Обработка чеков через QR и ФНС

!!! abstract "QR-flow"

    Система извлекает QR-код из загруженного изображения и получает
    официальный JSON чека через API ФНС. Далее данные валидируются,
    категоризируются и сохраняются в системе.

## Обзор технологии

### Используемые технологии

- **QR extractor** - извлечение строки QR из изображения чека
- **FNS API** - получение официального JSON документа по QR
- **Mapper + validator** - нормализация и валидация данных чека
- **Category service** - автокатегоризация позиций в чеке

### Архитектура системы

```python
def process_pending_receipt(pending):
    """Основная функция обработки изображения"""
    qr_data = QRCodeExtractor().extract(image_file)
    fns_payload = FNSClient().fetch_receipt(qr_data.raw)
    return map_fns_receipt_to_receipt_data(fns_payload)
```

## Процесс обработки

### Шаг 1: Загрузка изображения с QR-кодом

!!! tip "Поддерживаемые форматы"

    - **JPEG** (.jpg, .jpeg)
    - **PNG** (.png)
    - **Максимальный размер** - 10 МБ
    - На изображении должен быть читаемый QR-код чека

### Шаг 2: Извлечение QR-кода

```python
with pending.image_file.open('rb') as image_fp:
    qr_data = QRCodeExtractor().extract(image_fp)
```

### Шаг 3: Получение чека из ФНС

По сырой QR-строке создается тикет в ФНС, затем система опрашивает статус
и получает готовый документ чека:

```json
{
  "id": "ticket-id",
  "ticket": {
    "document": {
      "receipt": {
        "totalSum": 33500
      }
    }
  }
}
```

### Шаг 4: Маппинг и валидация

```python
receipt_data = map_fns_receipt_to_receipt_data(fns_payload)
validated = validate_receipt_parse_payload(receipt_data).to_dict()
```

## Извлекаемые данные

### Основная информация чека

- ✅ **Название продавца** (`name_seller`)
- ✅ **Дата и время** (`receipt_date`) - формат ДД.ММ.ГГГГ ЧЧ:ММ
- ✅ **Номер чека** (`number_receipt`) - числовое значение
- ✅ **Общая сумма** (`total_sum`) - с учетом копеек
- ✅ **Тип операции** (`operation_type`) - 1 (покупка) или 2 (возврат)

### Информация о продавце

- ✅ **Адрес расчетов** (`retail_place_address`)
- ✅ **Место расчетов** (`retail_place`)

### Налоговая информация

- ✅ **НДС 10%** (`nds10`) - сумма налога
- ✅ **НДС 22%** (`nds20`) - сумма налога

### Товары

Для каждого товара извлекается:

- ✅ **Название товара** (`product_name`)
- ✅ **Категория** (`category`) - автоматически определяется
- ✅ **Цена за единицу** (`price`)
- ✅ **Количество** (`quantity`)
- ✅ **Общая сумма** (`amount`) - цена × количество

## Надежность обработки

### Факторы, влияющие на результат

- **Качество QR-кода** - читаемость, отсутствие размытия/бликов
- **Доступность ФНС API** - ответы и лимиты сервиса ФНС
- **Корректность данных в QR** - валидный формат строки чека

### Практические ожидания

- При корректном QR, как правило, возвращается полный официальный документ.
- Если QR нечитабелен или поврежден, обработка завершается ошибкой.
- При временных сбоях ФНС доступна повторная попытка.

!!! warning "Ограничения"

    - Избегайте бликов и размытия в области QR-кода
    - Размещайте QR крупно и в фокусе
    - При ошибке попробуйте другой снимок QR

```bash

FNS_INN=123456789012
FNS_PASSWORD=your-fns-password
FNS_CLIENT_SECRET=your-fns-client-secret

# Опциональные таймауты/ретраи

FNS_TIMEOUT_SECONDS=10
FNS_POLL_ATTEMPTS=5
FNS_POLL_INTERVAL_SECONDS=1
FNS_SESSION_CACHE_TTL_SECONDS=3600
```

### Параметры интеграции

- **FNS_INN** - ИНН аккаунта ФНС
- **FNS_PASSWORD** - пароль аккаунта ФНС
- **FNS_CLIENT_SECRET** - client secret ФНС API
- **FNS_TIMEOUT_SECONDS** - таймаут HTTP-запросов
- **FNS_POLL_ATTEMPTS/FNS_POLL_INTERVAL_SECONDS** - поведение polling

## Обработка ошибок

### Типичные ошибки

#### "Не удалось распознать изображение"

**Причины:**
- Низкое качество изображения
- Неподдерживаемый формат
- Проблемы с чтением QR-кода
- Временная недоступность API ФНС

**Решение:**
- Убедитесь, что QR в фокусе
- Попробуйте другой снимок QR
- Добавьте чек вручную

#### "Не удалось получить корректный ответ ФНС"

**Причины:**
- Временный сбой в API ФНС
- Некорректный формат QR-строки

**Решение:**
- Проверьте доступ к API ФНС
- Попробуйте еще раз
- Обратитесь к администратору

### Логирование

```python
import logging
logger = logging.getLogger(__name__)

def process_receipt_image(image_file):
    try:
        qr_data = QRCodeExtractor().extract(image_file)
        logger.info('QR extracted successfully')

        fns_payload = FNSClient().fetch_receipt(qr_data.raw)
        logger.info('FNS payload loaded successfully')

        parsed_result = map_fns_receipt_to_receipt_data(fns_payload)
        logger.info('FNS payload mapped successfully')

        return parsed_result

    except Exception as e:
        logger.error(f"Receipt processing failed: {e}")
        raise
```
