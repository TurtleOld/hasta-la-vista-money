# Управление счетами и финансовая аналитика

Центральный модуль управления финансовыми счетами в приложении "Hasta la Vista, Money!" с встроенной аналитикой доходов и расходов.

## Обзор функциональности

### Основные возможности

- **Управление счетами** - создание, редактирование и удаление финансовых счетов
- **Переводы между счетами** - переводы средств с логированием операций
- **Финансовая аналитика** - графики доходов и расходов в реальном времени
- **Мультивалютность** - поддержка 8 валют (RUB, USD, EUR, GBP, CZK, PLN, TRY, CNH). Пока без автоматической конвертации между валютами.
- **Типы счетов** - кредитные, дебетовые счета и наличные
- **Журнал операций** - полная история переводов между счетами

### Структура данных

Каждый счет содержит следующую информацию:

- **Название** - уникальное имя счета (до 250 символов)
- **Тип** - кредитный, дебетовый или наличные
- **Баланс** - текущая сумма на счете (до 20 знаков)
- **Валюта** - валюта счета из поддерживаемого списка
- **Пользователь** - владелец счета
- **Дата создания** - автоматически заполняется

## Интерфейс

### Главная страница счетов

На главной странице управления счетами доступны функции для работы с группами счетов и просмотра общей информации.

#### Основные элементы интерфейса

- **Селектор "Группа счетов"** — выпадающий список для выбора группы, счета которой будут отображаться на странице. После выбора группы список счетов обновляются в соответствии с выбранной группой.
- **Кнопки управления**:
  - **Детальная статистика** — переход к подробной аналитике по доходам, расходам, чекам и переводам.
  - **Добавить счет** — открывает форму создания нового счета в выбранной группе.
  - **Перевести средства** — позволяет выполнить перевод между счетами (в рамках группы).
- **Карточки счетов** — каждый счет отображается отдельной карточкой с информацией:
  - Название счета
  - Тип счета (дебетовый, кредитный и т.д.)
  - Баланс и валюта
  - Владелец (если счет не ваш)
  - Кнопки действий: редактировать ✏️, удалить 🗑️
- **Блок "Информация"** — краткое описание возможностей страницы и ссылка на детальную статистику.
- **Блок "Сумма всех счетов вашей группы"** — отображает общую сумму по всем счетам всех пользователей группы, в которых состоит пользователь.

!!! info **Блок "Сумма всех счетов вашей группы"**  
Этот блок отображается всегда, если пользователь не состоит в группах, отображается сумма всех его счетов.

## Управление счетами

### Работа с группами счетов

- Для просмотра счетов определённой группы выберите нужную группу в селекторе "Группа счетов".
- Все действия (создание, редактирование, удаление счетов, переводы) выполняются только над своими счетами.

### Создание нового счета

!!! abstract "Пошаговая инструкция"

    1. Нажмите кнопку "Добавить счет".
    2. Заполните форму:
       - Название счета (обязательно)
       - Начальный баланс (обязательно)
       - Валюта (обязательно)
       - Тип счета (по умолчанию дебетовый)
    4. Нажмите "Добавить новый счёт".

### Переводы между счетами

!!! abstract "Процедура перевода"

    1. Нажмите кнопку "Перевести средства".
    2. Выберите счета списания и зачисления.
    3. Введите сумму, дату и (опционально) заметку.
    4. Нажмите "Выполнить перевод".

При следующем переводе счета списания и зачисления из последнего успешного
перевода будут выбраны автоматически. При необходимости выбор можно изменить.

### Просмотр и редактирование счетов

- Счета отображаются в виде карточек с основной информацией.
- Для редактирования счета нажмите на иконку ✏️ на карточке нужного счета.
- Для удаления счета нажмите на иконку 🗑️ (удаление доступно только для счетов с нулевым балансом).

## Финансовая аналитика

### График доходов и расходов

График автоматически собирает данные из всех модулей:

#### Источники данных

- **Расходы** - из модуля `expense`
- **Доходы** - из модуля `income`
- **Чеки** - автоматически конвертируются в расходы
- **Переводы** - не влияют на график (внутренние операции)

#### Обработка данных

```python
# Автоматическое объединение по датам
expense_dataset = Expense.objects.filter(user=user).values('date').annotate(
    total_amount=Sum('amount')
).order_by('date')

income_dataset = Income.objects.filter(user=user).values('date').annotate(
    total_amount=Sum('amount')
).order_by('date')
```

#### Особенности отображения

- **Уникальные даты** - дублирующиеся даты объединяются
- **Сортировка** - данные отображаются в хронологическом порядке
- **Масштабирование** - автоматическое подстраивание под данные
- **Интерактивность** - наведение показывает точные значения

### Детальная статистика

!!! abstract "Доступ к статистике"

    1. **Нажмите кнопку** "Детальная статистика"
    2. **Перейдите** на страницу подробной аналитики
    3. **Изучите** различные отчеты и графики

#### Доступные отчеты

- **Расходы по категориям** - структура трат
- **Доходы по источникам** - анализ поступлений
- **Месячная динамика** - динамика изменения
- **Сравнительная аналитика** - периоды vs периоды

## Журнал переводов

### Структура записи

Каждый перевод логируется с полной информацией:

```python
class TransferMoneyLog(models.Model):
    user = models.ForeignKey(User, on_delete=models.CASCADE)
    from_account = models.ForeignKey(Account, related_name='from_account')
    to_account = models.ForeignKey(Account, related_name='to_account')
    amount = models.DecimalField(max_digits=20, decimal_places=2)
    exchange_date = models.DateTimeField()
    notes = models.TextField(blank=True)
    created_at = models.DateTimeField(auto_now_add=True)
```

### Просмотр истории

- **Хронологический порядок** - новые переводы сверху
- **Детальная информация** - счета, суммы, даты, заметки
- **Фильтрация** - по счетам, периодам, суммам

### Пример записи

```text
20-12-2023 15:30. Перевод суммы 5000.00 со счёта "Основной счёт" на счёт "Сбережения".
```

## Безопасность и валидация

### Права доступа

- **Аутентификация** - все операции требуют входа
- **Изоляция данных** - пользователи видят только свои счета
- **Группировка счетов** - Пользователи могут объединиться в группу и видеть общую информацию по счетам, расходам и доходам.
- **Проверка владельца** - операции только с собственными счетами

### Валидация данных

```python
# Проверка достаточности средств
if amount > from_account.balance:
    raise ValidationError("Недостаточно средств на счете")

# Проверка различных счетов
if from_account == to_account:
    raise ValidationError("Нельзя переводить на тот же счет")
```

## Устранение неполадок

### Частые проблемы

#### "Недостаточно средств на счете"

**Причина:** Попытка перевода суммы, превышающей баланс.

**Решение:**

- Проверьте текущий баланс счета
- Уменьшите сумму перевода
- Пополните счет перед переводом

#### "Нельзя переводить на тот же счет"

**Причина:** Выбраны одинаковые счета для перевода.

**Решение:**

- Выберите разные счета
- Проверьте правильность выбора

#### "Счет не найден"

**Причина:** Счет был удален или недоступен.

**Решение:**

- Проверьте список доступных счетов
- Создайте новый счет при необходимости
- Обратитесь к администратору
